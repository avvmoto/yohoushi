Yohoushi

YohoushiはGrowthForecastと連携し動作するグラフツールです。Yohoushiを使うことで、GrowthForecastをグラフデータ格納とグラフ画像生成用のバックエンドサーバとして用い、Yohoushiではグラフビューア機能を提供します。

YohoushiではGrowthForecastでは実現できなかった様々なビューでグラフを確認できます。例えば、グラフを大きくしたり、from/toの日時指定でグラフを確認したり、サムネイルサイズでたくさんあるグラフを一覧で見たりすることできます。またGrowthForecastではグラフ階層が3階層固定でしたがYohoushiではこの制約を無くすことも可能です。

また複数のGrowthForecastを集約してユーザに見せることが可能です。この特徴によるメリットのとしてGrowthForecastサーバの台数を増やすことでシステムを容易にスケールできるようになります。また、たとえば各拠点に散らばっていたGrowthForecastサーバを1つのビューで見せることも可能です。

         +----------+                 +---+
         |          |            +--> |GF1| <--- post a number 
Ops ---> | Yohoushi | <----------+    +---+
         |          |            |    +---+
         +----------+            +--> |GF2| <--- post a number
                                      +---+
                                        .
                                        .

また、既に運用中のGrowthForecastがシステムに存在する場合には、既存のGrowthForecastのシステムに変更を加えることなくYohoushiをお試しできます。これはYohoushiがGrowthForecastのグラフやグラフパス一覧を参照するのみなので、たとえばcronなどで動作しているGrowthForecastへ値を登録するスクリプトを変更する必要はありません。

Installation

Requirements

まず以下のソフトウェアをインストールしてください。

  1. GrowthForecast (>=0.62)
  2. Ruby (>=2.0.0)
  3. Rails (>=4.0.0)< /li>
  4. MySQL or Sqlite3 (>=3.6.16)

Yohoushiはグラフパス名など保存するためにMySQLを使用します。MySQLの代わりにSQLite3を使うことも可能で、その場合にはcp config/database-sqlite.yml config/database.ymlのようにデータベース設定を変更してください。

Yohoushiのインストール

以下を実行すると、MySQLデータベース上にyohoushi_productionという名前のデータベースが作成されます。
  1. git clone git@github.com:yohoushi/yohoushi.git
  2. cd yohoushi
  3. bundle install
  4. bundle rake db:create
  5. bundle rake db:migrate

Configuration

GrowthForecastサーバのURLを設定ファイルに登録してください。config/application.ymlのmappingキー以下にURLを登録します。下の設定例ではYohoushiを起動するとhttp://gfserver.local:5125を参照し始めます。

defaults: &defaults
  multiforecast:
  mapping:
    '':     'http://gfserver.local:5125'
  tagcloud:
  limit: 400

Run

以下のコマンドでYohoushiが起動します。bundle execを前に付ける必要はありません。

$ bin/yohoushi

bin/yohoushiを実行すると、Webサーバ用workerとGrowthForecastのパス情報を定期的に参照するworkerが起動します。Ctrl+cで停止します。-dオプションを付けるとデーモン化します。

webブラウザでhttp://localhost:4804にアクセスしてみてください! Yohoushiの画面を確認できます。

なお、注意点として、初回起動時の起動直後はGrowthForecastのグラフパスデータの同期に時間がかかります。そのため同期が完了するまでの間はYohoushi上のグラフパスがGrowthForecastのものと異なるように見えますが、しばらく時間をあけると同期が完了します。2回目以降の同期処理は差分更新のため時間がかかることはありません。

デーモン化したYohoushiを停止するには以下を実行してください。

$ bin/yohoushi stop

For more information

Yohoushiを4階層以上のグラフを登録するにはmultiforecast-client gemを使ってデータをPOSTすることになります。
